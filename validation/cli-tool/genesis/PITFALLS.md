## Pitfall 1: shutil.move follows symlinks - moves target, not link

**Where:** https://bugs.python.org/issue37193
**Why:** `shutil.move("symlink_to_important_file", dest)` moves the actual file the symlink points to, not the symlink itself. A user running the organizer on a directory containing symlinks to system files can silently relocate files outside the source directory - including files they don't own.
**Mitigation:** Check `os.path.islink()` before any move. By default, skip symlinks and warn. Add `--follow-symlinks` flag only for explicit opt-in.
**mitigation_file_path:** src/core.py

**Implementation:**
- Create: src/core.py - `organize_files()` checks `file.is_symlink()` first; if True, skips and appends to `skipped` list with reason "symlink"
- Test: tests/test_core.py - create a symlink to a file outside the temp dir, verify organize_files does NOT move the target and does NOT follow the link
- Constrain: no `shutil.move` call without a preceding `os.path.islink()` check

## Pitfall 2: Silent overwrite when duplicate filename exists in destination

**Where:** https://github.com/pallets/click/issues/2416 (pattern: missing pre-move validation)
**Why:** `shutil.move("photo.jpg", "images/photo.jpg")` silently overwrites an existing `images/photo.jpg`. If the user runs the organizer twice (or has two files named `photo.jpg` in different subdirs), the second file overwrites the first with no warning. Data loss with no error.
**Mitigation:** Check if destination exists before moving. If it does: rename with counter suffix (`photo_2.jpg`, `photo_3.jpg`) or skip with warning. Never silently overwrite.
**mitigation_file_path:** src/core.py

**Implementation:**
- Create: `safe_destination(dest: Path) -> Path` in src/core.py that returns `dest` if it doesn't exist, else `dest.stem + "_2" + dest.suffix` (incrementing until free)
- Test: tests/test_core.py - pre-create `images/photo.jpg`, then run organize on a dir with another `photo.jpg`, verify BOTH files exist after (photo.jpg and photo_2.jpg), not one overwritten
- Constrain: no `shutil.move(src, dest)` without calling `safe_destination(dest)` first

## Pitfall 3: Path traversal via malformed filename or source argument

**Where:** https://github.com/pallets/click/issues/1846 (pattern: raw user input to filesystem ops)
**Why:** If the CLI accepts a directory argument and passes it directly to `Path(directory).iterdir()`, a user can pass `../../etc` or a directory containing files named `../secret.txt` (possible on some filesystems). The organizer then reads/moves files outside the intended scope.
**Mitigation:** Resolve and validate the source path before any operation. Reject paths that resolve outside the current working directory (or a configured base). All destination paths must be validated to stay inside source_dir.
**mitigation_file_path:** src/security.py

**Implementation:**
- Create: src/security.py with `get_safe_path(base: Path, target: str) -> Path` that calls `.resolve()` on both, then asserts target starts with base; raises `ValueError` if not
- Test: tests/test_security.py - pass `../../etc` as directory, verify ValueError raised before any filesystem op; pass valid subdir, verify it succeeds
- Constrain: no `Path(user_input)` without calling `get_safe_path` first in cli.py
