# Research Report: csv-transform-cli
Generated by Genesis Architect on 2026-05-07

## Executive Summary
The TypeScript CLI ecosystem has converged on Commander.js or Yargs for argument parsing,
with a strong preference for a single `src/core.ts` entry point that isolates business logic
from the CLI adapter. The top projects all separate transformation logic from I/O, making
testing straightforward without mocking the filesystem.

## Market Landscape
TypeScript CLI tools have two dominant patterns: Commander.js for simple flag-based tools,
and oclif for plugin-based CLIs. For a CSV transformation tool, Commander.js is the clear
choice - oclif adds unnecessary complexity. All top tools use streaming for file I/O to
handle large inputs without memory pressure.

## Architecture Decision Rationale
Chose the Minimalist tier with Commander.js based on 4 of 5 analyzed repos using this
approach for single-purpose transformation tools. The `src/core.ts` isolation pattern
appears in all top repos, making the transformation logic independently testable.
