# Research Report: batch-rename
Generated by Genesis Architect on 2026-05-07

## Executive Summary
The Python batch-rename ecosystem is dominated by tools like mnamer (~700 stars) that evolved organically and accumulated technical debt around sanitization, config loading, and pattern escaping. The ecosystem converges on `pathlib.Path` + typer/click + pyproject.toml. The most-requested missing features across all analyzed tools are dry-run mode, undo support, and copy-instead-of-move -- all built into this scaffold from day one.

## Market Landscape
The space is fragmented: many small tools (10-80 stars) with narrow feature sets, and one dominant tool (mnamer at ~700 stars) that is media-specific. No general-purpose Python CLI renamer with dry-run + undo + regex + counter as first-class features exists above 100 stars. The gap is real.

Ecosystem convergence:
- `pathlib.Path` universally preferred over `os.rename`
- `pyproject.toml` is the modern baseline (not setup.py)
- `typer` or `click` for CLI (typer trending due to type hints)
- `rich` for terminal output (tables, color)
- `pytest` for testing

## Architecture Decision Rationale
- `sanitizer.py` isolated as single source of truth - prevents the double-sanitization bug found in mnamer #266
- `patterns.py` always `re.escape()` by default - prevents the raw-regex crash found in mnamer #132
- `config.py` raises explicitly on missing `--config` path - prevents silent failure from mnamer #55
- `log.py` writes `.rename_log.json` after every run - enables undo (missing from all analyzed tools)
- `renamer.py` is a pure function (no I/O) - enables testing without real files on disk
