What makes it different

Not a template generator.

Genesis doesn't copy a folder structure — it synthesizes the ecosystem and extracts real failures from production projects.

🔬

Real GitHub Issue Scanning

Reads the last 50 issues per top repo — open and closed. Extracts recurring errors, architecture regrets, and security patches from production.

🧠

The Wise Average

Doesn't copy one project. Synthesizes the most common folder structure (ecosystem convergence) with the highest-starred repo's decisions (quality signal).

💨

Mandatory Smoke Test

Auto-detects entrypoint from pyproject.toml or package.json. Never announces "complete" without exit code 0. If it fails, it fixes it first.

⚠️

Deprecation Radar

Flags dependencies with last commit older than 18 months, archived repos, and migration-away patterns across analyzed projects.

📊

Research Quality Signal

Full / Partial / Thin rating shown before architecture choice. Fewer than 5 repos triggers a warning and offers alternatives.

🤝

Team Config Sharing

Generates .genesis.json after scaffolding. Teammates run genesis init --from-team-config to build the same scaffold without repeating research.

The workflow

8 phases. Sequential. Gated.

Never guesses on architecture decisions. Every step is declared before it runs.

Phase 0
Environment Probe
OS, Python, package manager. Windows PATH via sysconfig.
Phase 1
Vision Alignment
2–3 questions A/B/C/D. Waits for answers. Never guesses.
Phase 2
Deep Discovery
15–20 repos. 50 issues each. GitHub → Exa → Web fallback.
Phase 3
Architecture Analysis
The Wise Average. Language confirmation. Windows pitfall watchlist.
Phase 4
Pitfall Identification
3–7 real pitfalls: issue link + root cause + mitigation.
Phase 5
Interactive Choice
A: Minimalist / B: Scalable / C: Research decides / D: Hybrid.
Phase 6
Genesis Build
Structure → deps → boilerplate → tests → CI/CD → smoke test.
Phase 7
Companion Mode
Research partner for the whole project. Restores from RESEARCH.md.
