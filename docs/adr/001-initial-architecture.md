# ADR 001 - Genesis Architect Repository Structure

## Status
Accepted

## Context
Genesis Architect is a Claude Code skill, not a standard application. It has no `src/` package
because the deliverable is `SKILL.md` (a behavior definition file), not importable Python code.
Scripts are support utilities invoked by Claude during skill execution, not a library.

## Decision
Top-level directories in this repository:

- `scripts/` - support scripts invoked during skill phases (issue_miner, vault, resolve_engine, etc.)
- `tests/` - pytest suite for scripts
- `references/` - reference documents read by the skill (architecture-patterns, mcp-strategy, security-templates)
- `assets/` - template files for skill deliverables (RESEARCH, PITFALLS, ROADMAP templates)
- `examples/` - working scaffold examples generated by the skill
- `evals/` - evaluation harness for skill quality measurement
- `docs/` - architecture decision records (this file)

## Consequences
- `drift_detector.py` should use this ADR to establish the baseline for structural drift detection
- New top-level directories require an update to this ADR
